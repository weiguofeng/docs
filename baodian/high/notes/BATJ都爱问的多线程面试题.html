BATJйғҪзҲұй—®зҡ„еӨҡзәҝзЁӢйқўиҜ•йўҳ

й”Ӣж јblog 2020-12-19 йқўиҜ•

# дёҖ йқўиҜ•дёӯе…ідәҺ synchronized е…ій”®еӯ—зҡ„ 5 иҝһеҮ»

# 1.1 иҜҙдёҖиҜҙиҮӘе·ұеҜ№дәҺ synchronized е…ій”®еӯ—зҡ„дәҶи§Ј

synchronizedе…ій”®еӯ—и§ЈеҶізҡ„жҳҜеӨҡдёӘзәҝзЁӢд№Ӣй—ҙи®ҝй—®иө„жәҗзҡ„еҗҢжӯҘжҖ§пјҢsynchronizedе…ій”®еӯ—еҸҜд»ҘдҝқиҜҒиў«е®ғдҝ®йҘ°зҡ„ж–№жі•жҲ–иҖ…д»Јз Ғеқ—еңЁд»»ж„Ҹж—¶еҲ»еҸӘиғҪжңүдёҖдёӘзәҝзЁӢжү§иЎҢгҖӮ

еҸҰеӨ–пјҢеңЁ Java ж—©жңҹзүҲжң¬дёӯпјҢsynchronizedеұһдәҺйҮҚйҮҸзә§й”ҒпјҢж•ҲзҺҮдҪҺдёӢпјҢеӣ дёәзӣ‘и§ҶеҷЁй”ҒпјҲmonitorпјүжҳҜдҫқиө–дәҺеә•еұӮзҡ„ж“ҚдҪңзі»з»ҹзҡ„ Mutex Lock жқҘе®һзҺ°зҡ„пјҢJava зҡ„зәҝзЁӢжҳҜжҳ е°„еҲ°ж“ҚдҪңзі»з»ҹзҡ„еҺҹз”ҹзәҝзЁӢд№ӢдёҠзҡ„гҖӮеҰӮжһңиҰҒжҢӮиө·жҲ–иҖ…е”ӨйҶ’дёҖдёӘзәҝзЁӢпјҢйғҪйңҖиҰҒж“ҚдҪңзі»з»ҹеё®еҝҷе®ҢжҲҗпјҢиҖҢж“ҚдҪңзі»з»ҹе®һзҺ°зәҝзЁӢд№Ӣй—ҙзҡ„еҲҮжҚўж—¶йңҖиҰҒд»Һз”ЁжҲ·жҖҒиҪ¬жҚўеҲ°еҶ…ж ёжҖҒпјҢиҝҷдёӘзҠ¶жҖҒд№Ӣй—ҙзҡ„иҪ¬жҚўйңҖиҰҒзӣёеҜ№жҜ”иҫғй•ҝзҡ„ж—¶й—ҙпјҢж—¶й—ҙжҲҗжң¬зӣёеҜ№иҫғй«ҳпјҢиҝҷд№ҹжҳҜдёәд»Җд№Ҳж—©жңҹзҡ„ synchronized ж•ҲзҺҮдҪҺзҡ„еҺҹеӣ гҖӮеәҶе№ёзҡ„жҳҜеңЁ Java 6 д№ӢеҗҺ Java е®ҳж–№еҜ№д»Һ JVM еұӮйқўеҜ№synchronized иҫғеӨ§дјҳеҢ–пјҢжүҖд»ҘзҺ°еңЁзҡ„ synchronized й”Ғж•ҲзҺҮд№ҹдјҳеҢ–еҫ—еҫҲдёҚй”ҷдәҶгҖӮJDK1.6еҜ№й”Ғзҡ„е®һзҺ°еј•е…ҘдәҶеӨ§йҮҸзҡ„дјҳеҢ–пјҢеҰӮиҮӘж—Ӣй”ҒгҖҒйҖӮеә”жҖ§иҮӘж—Ӣй”ҒгҖҒй”Ғж¶ҲйҷӨгҖҒй”ҒзІ—еҢ–гҖҒеҒҸеҗ‘й”ҒгҖҒиҪ»йҮҸзә§й”ҒзӯүжҠҖжңҜжқҘеҮҸе°‘й”Ғж“ҚдҪңзҡ„ејҖй”ҖгҖӮ

# 1.2 иҜҙиҜҙиҮӘе·ұжҳҜжҖҺд№ҲдҪҝз”Ё synchronized е…ій”®еӯ—пјҢеңЁйЎ№зӣ®дёӯз”ЁеҲ°дәҶеҗ—

synchronizedе…ій”®еӯ—жңҖдё»иҰҒзҡ„дёүз§ҚдҪҝз”Ёж–№ејҸпјҡ

  • дҝ®йҘ°е®һдҫӢж–№жі•пјҢдҪңз”ЁдәҺеҪ“еүҚеҜ№иұЎе®һдҫӢеҠ й”ҒпјҢиҝӣе…ҘеҗҢжӯҘд»Јз ҒеүҚиҰҒиҺ·еҫ—еҪ“еүҚеҜ№иұЎе®һдҫӢзҡ„й”Ғ
  • дҝ®йҘ°йқҷжҖҒж–№жі•пјҢдҪңз”ЁдәҺеҪ“еүҚзұ»еҜ№иұЎеҠ й”ҒпјҢиҝӣе…ҘеҗҢжӯҘд»Јз ҒеүҚиҰҒиҺ·еҫ—еҪ“еүҚзұ»еҜ№иұЎзҡ„й”Ғ гҖӮд№ҹе°ұжҳҜз»ҷеҪ“еүҚзұ»еҠ й”ҒпјҢдјҡдҪңз”ЁдәҺзұ»зҡ„жүҖжңүеҜ№иұЎе®һдҫӢпјҢеӣ дёәйқҷжҖҒжҲҗе‘ҳдёҚеұһдәҺд»»дҪ•дёҖдёӘе®һдҫӢеҜ№иұЎпјҢжҳҜзұ»жҲҗе‘ҳпјҲ static иЎЁжҳҺиҝҷжҳҜиҜҘзұ»зҡ„дёҖдёӘйқҷжҖҒиө„жәҗпјҢдёҚз®ЎnewдәҶеӨҡе°‘дёӘеҜ№иұЎпјҢеҸӘжңүдёҖд»ҪпјҢжүҖд»ҘеҜ№иҜҘзұ»зҡ„жүҖжңүеҜ№иұЎйғҪеҠ дәҶй”ҒпјүгҖӮжүҖд»ҘеҰӮжһңдёҖдёӘзәҝзЁӢAи°ғз”ЁдёҖдёӘе®һдҫӢеҜ№иұЎзҡ„йқһйқҷжҖҒ synchronized ж–№жі•пјҢиҖҢзәҝзЁӢBйңҖиҰҒи°ғз”ЁиҝҷдёӘе®һдҫӢеҜ№иұЎжүҖеұһзұ»зҡ„йқҷжҖҒ synchronized ж–№жі•пјҢжҳҜе…Ғи®ёзҡ„пјҢдёҚдјҡеҸ‘з”ҹдә’ж–ҘзҺ°иұЎпјҢеӣ дёәи®ҝй—®йқҷжҖҒ synchronized ж–№жі•еҚ з”Ёзҡ„й”ҒжҳҜеҪ“еүҚзұ»зҡ„й”ҒпјҢиҖҢи®ҝй—®йқһйқҷжҖҒ synchronized ж–№жі•еҚ з”Ёзҡ„й”ҒжҳҜеҪ“еүҚе®һдҫӢеҜ№иұЎй”ҒгҖӮ
  • дҝ®йҘ°д»Јз Ғеқ—пјҢжҢҮе®ҡеҠ й”ҒеҜ№иұЎпјҢеҜ№з»ҷе®ҡеҜ№иұЎеҠ й”ҒпјҢиҝӣе…ҘеҗҢжӯҘд»Јз Ғеә“еүҚиҰҒиҺ·еҫ—з»ҷе®ҡеҜ№иұЎзҡ„й”ҒгҖӮ е’Ң synchronized ж–№жі•дёҖж ·пјҢsynchronized(this)д»Јз Ғеқ—д№ҹжҳҜй”Ғе®ҡеҪ“еүҚеҜ№иұЎзҡ„гҖӮsynchronized е…ій”®еӯ—еҠ еҲ° static йқҷжҖҒж–№жі•е’Ң synchronized(class)д»Јз Ғеқ—дёҠйғҪжҳҜжҳҜз»ҷ Class зұ»дёҠй”ҒгҖӮиҝҷйҮҢеҶҚжҸҗдёҖдёӢпјҡsynchronizedе…ій”®еӯ—еҠ еҲ°йқһ static йқҷжҖҒж–№жі•дёҠжҳҜз»ҷеҜ№иұЎе®һдҫӢдёҠй”ҒгҖӮеҸҰеӨ–йңҖиҰҒжіЁж„Ҹзҡ„жҳҜпјҡе°ҪйҮҸдёҚиҰҒдҪҝз”Ё synchronized(String a) еӣ дёәJVMдёӯпјҢеӯ—з¬ҰдёІеёёйҮҸжұ е…·жңүзј“еҶІеҠҹиғҪпјҒ

дёӢйқўжҲ‘е·ІдёҖдёӘеёёи§Ғзҡ„йқўиҜ•йўҳдёәдҫӢи®Іи§ЈдёҖдёӢ synchronized е…ій”®еӯ—зҡ„е…·дҪ“дҪҝз”ЁгҖӮ

йқўиҜ•дёӯйқўиҜ•е®ҳз»ҸеёёдјҡиҜҙпјҡвҖңеҚ•дҫӢжЁЎејҸдәҶи§Јеҗ—пјҹжқҘз»ҷжҲ‘жүӢеҶҷдёҖдёӢпјҒз»ҷжҲ‘и§ЈйҮҠдёҖдёӢеҸҢйҮҚжЈҖйӘҢй”Ғж–№ејҸе®һзҺ°еҚ•дҫӢжЁЎејҸзҡ„еҺҹзҗҶе‘—пјҒвҖқ

еҸҢйҮҚж ЎйӘҢй”Ғе®һзҺ°еҜ№иұЎеҚ•дҫӢпјҲзәҝзЁӢе®үе…Ёпјү

public class Singleton {

    private volatile static Singleton uniqueInstance;

    private Singleton() {
    }

    public static Singleton getUniqueInstance() {
       //е…ҲеҲӨж–ӯеҜ№иұЎжҳҜеҗҰе·Із»Ҹе®һдҫӢиҝҮпјҢжІЎжңүе®һдҫӢеҢ–иҝҮжүҚиҝӣе…ҘеҠ й”Ғд»Јз Ғ
        if (uniqueInstance == null) {
            //зұ»еҜ№иұЎеҠ й”Ғ
            synchronized (Singleton.class) {
                if (uniqueInstance == null) {
                    uniqueInstance = new Singleton();
                }
            }
        }
        return uniqueInstance;
    }
}

еҸҰеӨ–пјҢйңҖиҰҒжіЁж„Ҹ uniqueInstance йҮҮз”Ё volatile е…ій”®еӯ—дҝ®йҘ°д№ҹжҳҜеҫҲжңүеҝ…иҰҒгҖӮ

uniqueInstance йҮҮз”Ё volatile е…ій”®еӯ—дҝ®йҘ°д№ҹжҳҜеҫҲжңүеҝ…иҰҒзҡ„пјҢ uniqueInstance = new Singleton(); иҝҷж®өд»Јз Ғе…¶е®һжҳҜеҲҶдёәдёүжӯҘжү§иЎҢпјҡ

  1. дёә uniqueInstance еҲҶй…ҚеҶ…еӯҳз©әй—ҙ
  2. еҲқе§ӢеҢ– uniqueInstance
  3. е°Ҷ uniqueInstance жҢҮеҗ‘еҲҶй…Қзҡ„еҶ…еӯҳең°еқҖ

дҪҶжҳҜз”ұдәҺ JVM е…·жңүжҢҮд»ӨйҮҚжҺ’зҡ„зү№жҖ§пјҢжү§иЎҢйЎәеәҸжңүеҸҜиғҪеҸҳжҲҗ 1->3->2гҖӮжҢҮд»ӨйҮҚжҺ’еңЁеҚ•зәҝзЁӢзҺҜеўғдёӢдёҚдјҡеҮәе…Ҳй—®йўҳпјҢдҪҶжҳҜеңЁеӨҡзәҝзЁӢзҺҜеўғдёӢдјҡеҜјиҮҙдёҖдёӘзәҝзЁӢиҺ·еҫ—иҝҳжІЎжңүеҲқе§ӢеҢ–зҡ„е®һдҫӢгҖӮдҫӢеҰӮпјҢзәҝзЁӢ T1 жү§иЎҢдәҶ 1 е’Ң 3пјҢжӯӨж—¶ T2 и°ғз”Ё getUniqueInstance() еҗҺеҸ‘зҺ° uniqueInstance дёҚдёәз©әпјҢеӣ жӯӨиҝ”еӣһ uniqueInstanceпјҢдҪҶжӯӨж—¶ uniqueInstance иҝҳжңӘиў«еҲқе§ӢеҢ–гҖӮ

дҪҝз”Ё volatile еҸҜд»ҘзҰҒжӯў JVM зҡ„жҢҮд»ӨйҮҚжҺ’пјҢдҝқиҜҒеңЁеӨҡзәҝзЁӢзҺҜеўғдёӢд№ҹиғҪжӯЈеёёиҝҗиЎҢгҖӮ

# 1.5 и°Ҳи°Ҳ synchronizedе’ҢReenTrantLock зҡ„еҢәеҲ«

дёӨиҖ…йғҪжҳҜеҸҜйҮҚе…Ҙй”ҒгҖӮвҖңеҸҜйҮҚе…Ҙй”ҒвҖқжҰӮеҝөжҳҜпјҡиҮӘе·ұеҸҜд»ҘеҶҚж¬ЎиҺ·еҸ–иҮӘе·ұзҡ„еҶ…йғЁй”ҒгҖӮжҜ”еҰӮдёҖдёӘзәҝзЁӢиҺ·еҫ—дәҶжҹҗдёӘеҜ№иұЎзҡ„й”ҒпјҢжӯӨж—¶иҝҷдёӘеҜ№иұЎй”ҒиҝҳжІЎжңүйҮҠж”ҫпјҢеҪ“е…¶еҶҚж¬ЎжғіиҰҒиҺ·еҸ–иҝҷдёӘеҜ№иұЎзҡ„й”Ғзҡ„ж—¶еҖҷиҝҳжҳҜеҸҜд»ҘиҺ·еҸ–зҡ„пјҢеҰӮжһңдёҚеҸҜй”ҒйҮҚе…Ҙзҡ„иҜқпјҢе°ұдјҡйҖ жҲҗжӯ»й”ҒгҖӮеҗҢдёҖдёӘзәҝзЁӢжҜҸж¬ЎиҺ·еҸ–й”ҒпјҢй”Ғзҡ„и®Ўж•°еҷЁйғҪиҮӘеўһ1пјҢжүҖд»ҘиҰҒзӯүеҲ°й”Ғзҡ„и®Ўж•°еҷЁдёӢйҷҚдёә0ж—¶жүҚиғҪйҮҠж”ҫй”ҒгҖӮ

в‘Ў synchronized дҫқиө–дәҺ JVM иҖҢ ReenTrantLock дҫқиө–дәҺ API

synchronized жҳҜдҫқиө–дәҺ JVM е®һзҺ°зҡ„пјҢеүҚйқўжҲ‘д»¬д№ҹи®ІеҲ°дәҶ иҷҡжӢҹжңәеӣўйҳҹеңЁ JDK1.6 дёә synchronized е…ій”®еӯ—иҝӣиЎҢдәҶеҫҲеӨҡдјҳеҢ–пјҢдҪҶжҳҜиҝҷдәӣдјҳеҢ–йғҪжҳҜеңЁиҷҡжӢҹжңәеұӮйқўе®һзҺ°зҡ„пјҢе№¶жІЎжңүзӣҙжҺҘжҡҙйңІз»ҷжҲ‘д»¬гҖӮReenTrantLock жҳҜ JDK еұӮйқўе®һзҺ°зҡ„пјҲд№ҹе°ұжҳҜ API еұӮйқўпјҢйңҖиҰҒ lock() е’Ң unlock ж–№жі•й…ҚеҗҲ try/finally иҜӯеҸҘеқ—жқҘе®ҢжҲҗпјүпјҢжүҖд»ҘжҲ‘д»¬еҸҜд»ҘйҖҡиҝҮжҹҘзңӢе®ғзҡ„жәҗд»Јз ҒпјҢжқҘзңӢе®ғжҳҜеҰӮдҪ•е®һзҺ°зҡ„гҖӮ

в‘ў ReenTrantLock жҜ” synchronized еўһеҠ дәҶдёҖдәӣй«ҳзә§еҠҹиғҪ

зӣёжҜ”synchronizedпјҢReenTrantLockеўһеҠ дәҶдёҖдәӣй«ҳзә§еҠҹиғҪгҖӮдё»иҰҒжқҘиҜҙдё»иҰҒжңүдёүзӮ№пјҡв‘ зӯүеҫ…еҸҜдёӯж–ӯпјӣв‘ЎеҸҜе®һзҺ°е…¬е№ій”Ғпјӣв‘ўеҸҜе®һзҺ°йҖүжӢ©жҖ§йҖҡзҹҘпјҲй”ҒеҸҜд»Ҙз»‘е®ҡеӨҡдёӘжқЎд»¶пјү

  • ReenTrantLockжҸҗдҫӣдәҶдёҖз§ҚиғҪеӨҹдёӯж–ӯзӯүеҫ…й”Ғзҡ„зәҝзЁӢзҡ„жңәеҲ¶пјҢйҖҡиҝҮlock.lockInterruptibly()жқҘе®һзҺ°иҝҷдёӘжңәеҲ¶гҖӮд№ҹе°ұжҳҜиҜҙжӯЈеңЁзӯүеҫ…зҡ„зәҝзЁӢеҸҜд»ҘйҖүжӢ©ж”ҫејғзӯүеҫ…пјҢж”№дёәеӨ„зҗҶе…¶д»–дәӢжғ…гҖӮ
  • ReenTrantLockеҸҜд»ҘжҢҮе®ҡжҳҜе…¬е№ій”ҒиҝҳжҳҜйқһе…¬е№ій”ҒгҖӮиҖҢsynchronizedеҸӘиғҪжҳҜйқһе…¬е№ій”ҒгҖӮжүҖи°“зҡ„е…¬е№ій”Ғе°ұжҳҜе…Ҳзӯүеҫ…зҡ„зәҝзЁӢе…ҲиҺ·еҫ—й”ҒгҖӮ ReenTrantLockй»ҳи®Өжғ…еҶөжҳҜйқһе…¬е№ізҡ„пјҢеҸҜд»ҘйҖҡиҝҮ ReenTrantLockзұ»зҡ„ReentrantLock(boolean fair)жһ„йҖ ж–№жі•жқҘеҲ¶е®ҡжҳҜеҗҰжҳҜе…¬е№ізҡ„гҖӮ
  • synchronizedе…ій”®еӯ—дёҺwait()е’Ңnotify/notifyAll()ж–№жі•зӣёз»“еҗҲеҸҜд»Ҙе®һзҺ°зӯүеҫ…/йҖҡзҹҘжңәеҲ¶пјҢReentrantLockзұ»еҪ“з„¶д№ҹеҸҜд»Ҙе®һзҺ°пјҢдҪҶжҳҜйңҖиҰҒеҖҹеҠ©дәҺConditionжҺҘеҸЈдёҺnewCondition() ж–№жі•гҖӮConditionжҳҜJDK1.5д№ӢеҗҺжүҚжңүзҡ„пјҢе®ғе…·жңүеҫҲеҘҪзҡ„зҒөжҙ»жҖ§пјҢжҜ”еҰӮеҸҜд»Ҙе®һзҺ°еӨҡи·ҜйҖҡзҹҘеҠҹиғҪд№ҹе°ұжҳҜеңЁдёҖдёӘLockеҜ№иұЎдёӯеҸҜд»ҘеҲӣе»әеӨҡдёӘConditionе®һдҫӢпјҲеҚіеҜ№иұЎзӣ‘и§ҶеҷЁпјүпјҢзәҝзЁӢеҜ№иұЎеҸҜд»ҘжіЁеҶҢеңЁжҢҮе®ҡзҡ„ConditionдёӯпјҢд»ҺиҖҢеҸҜд»ҘжңүйҖүжӢ©жҖ§зҡ„иҝӣиЎҢзәҝзЁӢйҖҡзҹҘпјҢеңЁи°ғеәҰзәҝзЁӢдёҠжӣҙеҠ зҒөжҙ»гҖӮ еңЁдҪҝз”Ёnotify/notifyAll()ж–№жі•иҝӣиЎҢйҖҡзҹҘж—¶пјҢиў«йҖҡзҹҘзҡ„зәҝзЁӢжҳҜз”ұ JVM йҖүжӢ©зҡ„пјҢз”ЁReentrantLockзұ»з»“еҗҲConditionе®һдҫӢеҸҜд»Ҙе®һзҺ°вҖңйҖүжӢ©жҖ§йҖҡзҹҘвҖқ пјҢиҝҷдёӘеҠҹиғҪйқһеёёйҮҚиҰҒпјҢиҖҢдё”жҳҜConditionжҺҘеҸЈй»ҳи®ӨжҸҗдҫӣзҡ„гҖӮиҖҢsynchronizedе…ій”®еӯ—е°ұзӣёеҪ“дәҺж•ҙдёӘLockеҜ№иұЎдёӯеҸӘжңүдёҖдёӘConditionе®һдҫӢпјҢжүҖжңүзҡ„зәҝзЁӢйғҪжіЁеҶҢеңЁе®ғдёҖдёӘиә«дёҠгҖӮеҰӮжһңжү§иЎҢnotifyAll()ж–№жі•зҡ„иҜқе°ұдјҡйҖҡзҹҘжүҖжңүеӨ„дәҺзӯүеҫ…зҠ¶жҖҒзҡ„зәҝзЁӢиҝҷж ·дјҡйҖ жҲҗеҫҲеӨ§зҡ„ж•ҲзҺҮй—®йўҳпјҢиҖҢConditionе®һдҫӢзҡ„signalAll()ж–№жі• еҸӘдјҡе”ӨйҶ’жіЁеҶҢеңЁиҜҘConditionе®һдҫӢдёӯзҡ„жүҖжңүзӯүеҫ…зәҝзЁӢгҖӮ

  • volatileе…ій”®еӯ—жҳҜзәҝзЁӢеҗҢжӯҘзҡ„иҪ»йҮҸзә§е®һзҺ°пјҢжүҖд»ҘvolatileжҖ§иғҪиӮҜе®ҡжҜ”synchronizedе…ій”®еӯ—иҰҒеҘҪгҖӮдҪҶжҳҜvolatileе…ій”®еӯ—еҸӘиғҪз”ЁдәҺеҸҳйҮҸиҖҢsynchronizedе…ій”®еӯ—еҸҜд»Ҙдҝ®йҘ°ж–№жі•д»ҘеҸҠд»Јз Ғеқ—гҖӮsynchronizedе…ій”®еӯ—еңЁJavaSE1.6д№ӢеҗҺиҝӣиЎҢдәҶдё»иҰҒеҢ…жӢ¬дёәдәҶеҮҸе°‘иҺ·еҫ—й”Ғе’ҢйҮҠж”ҫй”ҒеёҰжқҘзҡ„жҖ§иғҪж¶ҲиҖ—иҖҢеј•е…Ҙзҡ„еҒҸеҗ‘й”Ғе’ҢиҪ»йҮҸзә§й”Ғд»ҘеҸҠе…¶е®ғеҗ„з§ҚдјҳеҢ–д№ӢеҗҺжү§иЎҢж•ҲзҺҮжңүдәҶжҳҫи‘—жҸҗеҚҮпјҢе®һйҷ…ејҖеҸ‘дёӯдҪҝз”Ё synchronized е…ій”®еӯ—зҡ„еңәжҷҜиҝҳжҳҜжӣҙеӨҡдёҖдәӣгҖӮ
  • еӨҡзәҝзЁӢи®ҝй—®volatileе…ій”®еӯ—дёҚдјҡеҸ‘з”ҹйҳ»еЎһпјҢиҖҢsynchronizedе…ій”®еӯ—еҸҜиғҪдјҡеҸ‘з”ҹйҳ»еЎһ
  • volatileе…ій”®еӯ—иғҪдҝқиҜҒж•°жҚ®зҡ„еҸҜи§ҒжҖ§пјҢдҪҶдёҚиғҪдҝқиҜҒж•°жҚ®зҡ„еҺҹеӯҗжҖ§гҖӮsynchronizedе…ій”®еӯ—дёӨиҖ…йғҪиғҪдҝқиҜҒгҖӮ
  • volatileе…ій”®еӯ—дё»иҰҒз”ЁдәҺи§ЈеҶіеҸҳйҮҸеңЁеӨҡдёӘзәҝзЁӢд№Ӣй—ҙзҡ„еҸҜи§ҒжҖ§пјҢиҖҢ synchronizedе…ій”®еӯ—и§ЈеҶізҡ„жҳҜеӨҡдёӘзәҝзЁӢд№Ӣй—ҙи®ҝй—®иө„жәҗзҡ„еҗҢжӯҘжҖ§гҖӮ

# дёү йқўиҜ•дёӯе…ідәҺ зәҝзЁӢжұ зҡ„ 2 иҝһеҮ»

# 3.1 дёәд»Җд№ҲиҰҒз”ЁзәҝзЁӢжұ пјҹ

зәҝзЁӢжұ жҸҗдҫӣдәҶдёҖз§ҚйҷҗеҲ¶е’Ңз®ЎзҗҶиө„жәҗпјҲеҢ…жӢ¬жү§иЎҢдёҖдёӘд»»еҠЎпјүгҖӮ жҜҸдёӘзәҝзЁӢжұ иҝҳз»ҙжҠӨдёҖдәӣеҹәжң¬з»ҹи®ЎдҝЎжҒҜпјҢдҫӢеҰӮе·Іе®ҢжҲҗд»»еҠЎзҡ„ж•°йҮҸгҖӮ

иҝҷйҮҢеҖҹз”ЁгҖҠJavaе№¶еҸ‘зј–зЁӢзҡ„иүәжңҜгҖӢжҸҗеҲ°зҡ„жқҘиҜҙдёҖдёӢдҪҝз”ЁзәҝзЁӢжұ зҡ„еҘҪеӨ„пјҡ

  • йҷҚдҪҺиө„жәҗж¶ҲиҖ—гҖӮ йҖҡиҝҮйҮҚеӨҚеҲ©з”Ёе·ІеҲӣе»әзҡ„зәҝзЁӢйҷҚдҪҺзәҝзЁӢеҲӣе»әе’Ңй”ҖжҜҒйҖ жҲҗзҡ„ж¶ҲиҖ—гҖӮ
  • жҸҗй«ҳе“Қеә”йҖҹеәҰгҖӮ еҪ“д»»еҠЎеҲ°иҫҫж—¶пјҢд»»еҠЎеҸҜд»ҘдёҚйңҖиҰҒзҡ„зӯүеҲ°зәҝзЁӢеҲӣе»әе°ұиғҪз«ӢеҚіжү§иЎҢгҖӮ
  • жҸҗй«ҳзәҝзЁӢзҡ„еҸҜз®ЎзҗҶжҖ§гҖӮ зәҝзЁӢжҳҜзЁҖзјәиө„жәҗпјҢеҰӮжһңж— йҷҗеҲ¶зҡ„еҲӣе»әпјҢдёҚд»…дјҡж¶ҲиҖ—зі»з»ҹиө„жәҗпјҢиҝҳдјҡйҷҚдҪҺзі»з»ҹзҡ„зЁіе®ҡжҖ§пјҢдҪҝз”ЁзәҝзЁӢжұ еҸҜд»ҘиҝӣиЎҢз»ҹдёҖзҡ„еҲҶй…ҚпјҢи°ғдјҳе’Ңзӣ‘жҺ§гҖӮ

# 3.2 е®һзҺ°RunnableжҺҘеҸЈе’ҢCallableжҺҘеҸЈзҡ„еҢәеҲ«

еҰӮжһңжғіи®©зәҝзЁӢжұ жү§иЎҢд»»еҠЎзҡ„иҜқйңҖиҰҒе®һзҺ°зҡ„RunnableжҺҘеҸЈжҲ–CallableжҺҘеҸЈгҖӮ RunnableжҺҘеҸЈжҲ–CallableжҺҘеҸЈе®һзҺ°зұ»йғҪеҸҜд»Ҙиў«ThreadPoolExecutorжҲ–ScheduledThreadPoolExecutorжү§иЎҢгҖӮдёӨиҖ…зҡ„еҢәеҲ«еңЁдәҺ Runnable жҺҘеҸЈдёҚдјҡиҝ”еӣһз»“жһңдҪҶжҳҜ Callable жҺҘеҸЈеҸҜд»Ҙиҝ”еӣһз»“жһңгҖӮ

еӨҮжіЁпјҡ е·Ҙе…·зұ»ExecutorsеҸҜд»Ҙе®һзҺ°RunnableеҜ№иұЎе’ҢCallableеҜ№иұЎд№Ӣй—ҙзҡ„зӣёдә’иҪ¬жҚўгҖӮпјҲExecutors.callableпјҲRunnable taskпјүжҲ–Executors.callableпјҲRunnable taskпјҢObject resuleпјүпјүгҖӮ

# 3.3 жү§иЎҢexecute()ж–№жі•е’Ңsubmit()ж–№жі•зҡ„еҢәеҲ«жҳҜд»Җд№Ҳе‘ўпјҹ

1)execute() ж–№жі•з”ЁдәҺжҸҗдәӨдёҚйңҖиҰҒиҝ”еӣһеҖјзҡ„д»»еҠЎпјҢжүҖд»Ҙж— жі•еҲӨж–ӯд»»еҠЎжҳҜеҗҰиў«зәҝзЁӢжұ жү§иЎҢжҲҗеҠҹдёҺеҗҰпјӣ

2)submit()ж–№жі•з”ЁдәҺжҸҗдәӨйңҖиҰҒиҝ”еӣһеҖјзҡ„д»»еҠЎгҖӮзәҝзЁӢжұ дјҡиҝ”еӣһдёҖдёӘfutureзұ»еһӢзҡ„еҜ№иұЎпјҢйҖҡиҝҮиҝҷдёӘfutureеҜ№иұЎеҸҜд»ҘеҲӨж–ӯд»»еҠЎжҳҜеҗҰжү§иЎҢжҲҗеҠҹпјҢе№¶дё”еҸҜд»ҘйҖҡиҝҮfutureзҡ„get()ж–№жі•жқҘиҺ·еҸ–иҝ”еӣһеҖјпјҢget()ж–№жі•дјҡйҳ»еЎһеҪ“еүҚзәҝзЁӢзӣҙеҲ°д»»еҠЎе®ҢжҲҗпјҢиҖҢдҪҝз”Ё getпјҲlong timeoutпјҢTimeUnit unitпјүж–№жі•еҲҷдјҡйҳ»еЎһеҪ“еүҚзәҝзЁӢдёҖж®өж—¶й—ҙеҗҺз«ӢеҚіиҝ”еӣһпјҢиҝҷж—¶еҖҷжңүеҸҜиғҪд»»еҠЎжІЎжңүжү§иЎҢе®ҢгҖӮ

# 3.4 еҰӮдҪ•еҲӣе»әзәҝзЁӢжұ 

гҖҠйҳҝйҮҢе·ҙе·ҙJavaејҖеҸ‘жүӢеҶҢгҖӢдёӯејәеҲ¶зәҝзЁӢжұ дёҚе…Ғи®ёдҪҝз”Ё Executors еҺ»еҲӣе»әпјҢиҖҢжҳҜйҖҡиҝҮ ThreadPoolExecutor зҡ„ж–№ејҸпјҢиҝҷж ·зҡ„еӨ„зҗҶж–№ејҸи®©еҶҷзҡ„еҗҢеӯҰжӣҙеҠ жҳҺзЎ®зәҝзЁӢжұ зҡ„иҝҗиЎҢи§„еҲҷпјҢи§„йҒҝиө„жәҗиҖ—е°Ҫзҡ„йЈҺйҷ©

Executors иҝ”еӣһзәҝзЁӢжұ еҜ№иұЎзҡ„ејҠз«ҜеҰӮдёӢпјҡ

  • FixedThreadPool е’Ң SingleThreadExecutor пјҡ е…Ғи®ёиҜ·жұӮзҡ„йҳҹеҲ—й•ҝеәҰдёә Integer.MAX_VALUE,еҸҜиғҪе Ҷз§ҜеӨ§йҮҸзҡ„иҜ·жұӮпјҢд»ҺиҖҢеҜјиҮҙOOMгҖӮ
  • CachedThreadPool е’Ң ScheduledThreadPool пјҡ е…Ғи®ёеҲӣе»әзҡ„зәҝзЁӢж•°йҮҸдёә Integer.MAX_VALUE пјҢеҸҜиғҪдјҡеҲӣе»әеӨ§йҮҸзәҝзЁӢпјҢд»ҺиҖҢеҜјиҮҙOOMгҖӮ

ж–№ејҸдёҖпјҡйҖҡиҝҮжһ„йҖ ж–№жі•е®һзҺ° ж–№ејҸдәҢпјҡйҖҡиҝҮExecutor жЎҶжһ¶зҡ„е·Ҙе…·зұ»ExecutorsжқҘе®һзҺ° жҲ‘д»¬еҸҜд»ҘеҲӣе»әдёүз§Қзұ»еһӢзҡ„ThreadPoolExecutorпјҡ

  • FixedThreadPool пјҡ иҜҘж–№жі•иҝ”еӣһдёҖдёӘеӣәе®ҡзәҝзЁӢж•°йҮҸзҡ„зәҝзЁӢжұ гҖӮиҜҘзәҝзЁӢжұ дёӯзҡ„зәҝзЁӢж•°йҮҸе§Ӣз»ҲдёҚеҸҳгҖӮеҪ“жңүдёҖдёӘж–°зҡ„д»»еҠЎжҸҗдәӨж—¶пјҢзәҝзЁӢжұ дёӯиӢҘжңүз©әй—ІзәҝзЁӢпјҢеҲҷз«ӢеҚіжү§иЎҢгҖӮиӢҘжІЎжңүпјҢеҲҷж–°зҡ„д»»еҠЎдјҡиў«жҡӮеӯҳеңЁдёҖдёӘд»»еҠЎйҳҹеҲ—дёӯпјҢеҫ…жңүзәҝзЁӢз©әй—Іж—¶пјҢдҫҝеӨ„зҗҶеңЁд»»еҠЎйҳҹеҲ—дёӯзҡ„д»»еҠЎгҖӮ
  • SingleThreadExecutorпјҡ ж–№жі•иҝ”еӣһдёҖдёӘеҸӘжңүдёҖдёӘзәҝзЁӢзҡ„зәҝзЁӢжұ гҖӮиӢҘеӨҡдҪҷдёҖдёӘд»»еҠЎиў«жҸҗдәӨеҲ°иҜҘзәҝзЁӢжұ пјҢд»»еҠЎдјҡиў«дҝқеӯҳеңЁдёҖдёӘд»»еҠЎйҳҹеҲ—дёӯпјҢеҫ…зәҝзЁӢз©әй—ІпјҢжҢүе…Ҳе…Ҙе…ҲеҮәзҡ„йЎәеәҸжү§иЎҢйҳҹеҲ—дёӯзҡ„д»»еҠЎгҖӮ
  • CachedThreadPoolпјҡ иҜҘж–№жі•иҝ”еӣһдёҖдёӘеҸҜж №жҚ®е®һйҷ…жғ…еҶөи°ғж•ҙзәҝзЁӢж•°йҮҸзҡ„зәҝзЁӢжұ гҖӮзәҝзЁӢжұ зҡ„зәҝзЁӢж•°йҮҸдёҚзЎ®е®ҡпјҢдҪҶиӢҘжңүз©әй—ІзәҝзЁӢеҸҜд»ҘеӨҚз”ЁпјҢеҲҷдјҡдјҳе…ҲдҪҝз”ЁеҸҜеӨҚз”Ёзҡ„зәҝзЁӢгҖӮиӢҘжүҖжңүзәҝзЁӢеқҮеңЁе·ҘдҪңпјҢеҸҲжңүж–°зҡ„д»»еҠЎжҸҗдәӨпјҢеҲҷдјҡеҲӣе»әж–°зҡ„зәҝзЁӢеӨ„зҗҶд»»еҠЎгҖӮжүҖжңүзәҝзЁӢеңЁеҪ“еүҚд»»еҠЎжү§иЎҢе®ҢжҜ•еҗҺпјҢе°Ҷиҝ”еӣһзәҝзЁӢжұ иҝӣиЎҢеӨҚз”ЁгҖӮ

еҜ№еә”Executorsе·Ҙе…·зұ»дёӯзҡ„ж–№жі•еҰӮеӣҫжүҖзӨәпјҡ

# еӣӣ йқўиҜ•дёӯе…ідәҺ Atomic еҺҹеӯҗзұ»зҡ„ 4 иҝһеҮ»

# 4.1 д»Ӣз»ҚдёҖдёӢAtomic еҺҹеӯҗзұ»

Atomic зҝ»иҜ‘жҲҗдёӯж–ҮжҳҜеҺҹеӯҗзҡ„ж„ҸжҖқгҖӮеңЁеҢ–еӯҰдёҠпјҢжҲ‘д»¬зҹҘйҒ“еҺҹеӯҗжҳҜжһ„жҲҗдёҖиҲ¬зү©иҙЁзҡ„жңҖе°ҸеҚ•дҪҚпјҢеңЁеҢ–еӯҰеҸҚеә”дёӯжҳҜдёҚеҸҜеҲҶеүІзҡ„гҖӮеңЁжҲ‘д»¬иҝҷйҮҢ Atomic жҳҜжҢҮдёҖдёӘж“ҚдҪңжҳҜдёҚеҸҜдёӯж–ӯзҡ„гҖӮеҚідҪҝжҳҜеңЁеӨҡдёӘзәҝзЁӢдёҖиө·жү§иЎҢзҡ„ж—¶еҖҷпјҢдёҖдёӘж“ҚдҪңдёҖж—ҰејҖе§ӢпјҢе°ұдёҚдјҡиў«е…¶д»–зәҝзЁӢе№Іжү°гҖӮ

жүҖд»ҘпјҢжүҖи°“еҺҹеӯҗзұ»иҜҙз®ҖеҚ•зӮ№е°ұжҳҜе…·жңүеҺҹеӯҗ/еҺҹеӯҗж“ҚдҪңзү№еҫҒзҡ„зұ»гҖӮ

е№¶еҸ‘еҢ… java.util.concurrent зҡ„еҺҹеӯҗзұ»йғҪеӯҳж”ҫеңЁjava.util.concurrent.atomicдёӢ,еҰӮдёӢеӣҫжүҖзӨәгҖӮ

# 4.2 JUC еҢ…дёӯзҡ„еҺҹеӯҗзұ»жҳҜе“Ә4зұ»?

еҹәжң¬зұ»еһӢ

дҪҝз”ЁеҺҹеӯҗзҡ„ж–№ејҸжӣҙж–°еҹәжң¬зұ»еһӢ

  • AtomicIntegerпјҡж•ҙеҪўеҺҹеӯҗзұ»
  • AtomicLongпјҡй•ҝж•ҙеһӢеҺҹеӯҗзұ»
  • AtomicBoolean пјҡеёғе°”еһӢеҺҹеӯҗзұ»

ж•°з»„зұ»еһӢ

дҪҝз”ЁеҺҹеӯҗзҡ„ж–№ејҸжӣҙж–°ж•°з»„йҮҢзҡ„жҹҗдёӘе…ғзҙ 

  • AtomicIntegerArrayпјҡж•ҙеҪўж•°з»„еҺҹеӯҗзұ»
  • AtomicLongArrayпјҡй•ҝж•ҙеҪўж•°з»„еҺҹеӯҗзұ»
  • AtomicReferenceArray пјҡеј•з”Ёзұ»еһӢж•°з»„еҺҹеӯҗзұ»

еј•з”Ёзұ»еһӢ

  • AtomicReferenceпјҡеј•з”Ёзұ»еһӢеҺҹеӯҗзұ»
  • AtomicStampedRerenceпјҡеҺҹеӯҗжӣҙж–°еј•з”Ёзұ»еһӢйҮҢзҡ„еӯ—ж®өеҺҹеӯҗзұ»
  • AtomicMarkableReference пјҡеҺҹеӯҗжӣҙж–°еёҰжңүж Үи®°дҪҚзҡ„еј•з”Ёзұ»еһӢ

еҜ№иұЎзҡ„еұһжҖ§дҝ®ж”№зұ»еһӢ

  • AtomicIntegerFieldUpdater:еҺҹеӯҗжӣҙж–°ж•ҙеҪўеӯ—ж®өзҡ„жӣҙж–°еҷЁ
  • AtomicLongFieldUpdaterпјҡеҺҹеӯҗжӣҙж–°й•ҝж•ҙеҪўеӯ—ж®өзҡ„жӣҙж–°еҷЁ
  • AtomicStampedReference пјҡеҺҹеӯҗжӣҙж–°еёҰжңүзүҲжң¬еҸ·зҡ„еј•з”Ёзұ»еһӢгҖӮиҜҘзұ»е°Ҷж•ҙж•°еҖјдёҺеј•з”Ёе…іиҒ”иө·жқҘпјҢеҸҜз”ЁдәҺи§ЈеҶіеҺҹеӯҗзҡ„жӣҙж–°ж•°жҚ®е’Ңж•°жҚ®зҡ„зүҲжң¬еҸ·пјҢеҸҜд»Ҙи§ЈеҶідҪҝз”Ё CAS иҝӣиЎҢеҺҹеӯҗжӣҙж–°ж—¶еҸҜиғҪеҮәзҺ°зҡ„ ABA й—®йўҳгҖӮ

# 5.3 AQS з»„д»¶жҖ»з»“

  • Semaphore(дҝЎеҸ·йҮҸ)-е…Ғи®ёеӨҡдёӘзәҝзЁӢеҗҢж—¶и®ҝй—®пјҡ synchronized е’Ң ReentrantLock йғҪжҳҜдёҖж¬ЎеҸӘе…Ғи®ёдёҖдёӘзәҝзЁӢи®ҝй—®жҹҗдёӘиө„жәҗпјҢSemaphore(дҝЎеҸ·йҮҸ)еҸҜд»ҘжҢҮе®ҡеӨҡдёӘзәҝзЁӢеҗҢж—¶и®ҝй—®жҹҗдёӘиө„жәҗгҖӮ
  • CountDownLatch пјҲеҖ’и®Ўж—¶еҷЁпјүпјҡ CountDownLatchжҳҜдёҖдёӘеҗҢжӯҘе·Ҙе…·зұ»пјҢз”ЁжқҘеҚҸи°ғеӨҡдёӘзәҝзЁӢд№Ӣй—ҙзҡ„еҗҢжӯҘгҖӮиҝҷдёӘе·Ҙе…·йҖҡеёёз”ЁжқҘжҺ§еҲ¶зәҝзЁӢзӯүеҫ…пјҢе®ғеҸҜд»Ҙи®©жҹҗдёҖдёӘзәҝзЁӢзӯүеҫ…зӣҙеҲ°еҖ’и®Ўж—¶з»“жқҹпјҢеҶҚејҖе§Ӣжү§иЎҢгҖӮ
  • CyclicBarrier(еҫӘзҺҜж …ж Ҹ)пјҡ CyclicBarrier е’Ң CountDownLatch йқһеёёзұ»дјјпјҢе®ғд№ҹеҸҜд»Ҙе®һзҺ°зәҝзЁӢй—ҙзҡ„жҠҖжңҜзӯүеҫ…пјҢдҪҶжҳҜе®ғзҡ„еҠҹиғҪжҜ” CountDownLatch жӣҙеҠ еӨҚжқӮе’ҢејәеӨ§гҖӮдё»иҰҒеә”з”ЁеңәжҷҜе’Ң CountDownLatch зұ»дјјгҖӮCyclicBarrier зҡ„еӯ—йқўж„ҸжҖқжҳҜеҸҜеҫӘзҺҜдҪҝз”ЁпјҲCyclicпјүзҡ„еұҸйҡңпјҲBarrierпјүгҖӮе®ғиҰҒеҒҡзҡ„дәӢжғ…жҳҜпјҢи®©дёҖз»„зәҝзЁӢеҲ°иҫҫдёҖдёӘеұҸйҡңпјҲд№ҹеҸҜд»ҘеҸ«еҗҢжӯҘзӮ№пјүж—¶иў«йҳ»еЎһпјҢзӣҙеҲ°жңҖеҗҺдёҖдёӘзәҝзЁӢеҲ°иҫҫеұҸйҡңж—¶пјҢеұҸйҡңжүҚдјҡејҖй—ЁпјҢжүҖжңүиў«еұҸйҡңжӢҰжҲӘзҡ„зәҝзЁӢжүҚдјҡз»§з»ӯе№Іжҙ»гҖӮCyclicBarrierй»ҳи®Өзҡ„жһ„йҖ ж–№жі•жҳҜ CyclicBarrier(int parties)пјҢе…¶еҸӮж•°иЎЁзӨәеұҸйҡңжӢҰжҲӘзҡ„зәҝзЁӢж•°йҮҸпјҢжҜҸдёӘзәҝзЁӢи°ғз”Ёawaitж–№жі•е‘ҠиҜү CyclicBarrier жҲ‘е·Із»ҸеҲ°иҫҫдәҶеұҸйҡңпјҢз„¶еҗҺеҪ“еүҚзәҝзЁӢиў«йҳ»еЎһгҖӮ
